A json API is available through /api/, or use the graphical interface to discover some pfams database tables and apply some filters (access to /admin).

Check the next example:

  • /api/PfamA?q={"filters":[{"name":"pfamA_id","op":"eq","val":"Piwi"}],"single":true}

There are some special queries that join multiple tables and accept some specific parameters. This queries respond with a json dictionary with the "query" and "output" keys.

First, it is posible to obtain a list of pfamA registers from a uniprot_id (eg. /api/query/pfam_uniprot/egfr_human).

Also, it is available a way to recover a list of PDB registers from a "sequence description" (eg. /api/query/pdb_sequencedescription/egfr_human,57,168) where the 3 parameters are a uniprot_id, a seq_start and seq_end inside that uniprot register.

Then, it is accesible an alternative to obtain a small PDB image (base 64 enconded after compressed) from an specific pdb_id (eg. /api/query/pdbimage_pdb/1IVO).

Last, it is posible to obtain an MSA in stockholm format through a pfamA_acc or pfamA_id (eg. /api/query/stockholm_pfam/piwi). The output value is zip compressed and then base 64 encoded, to optimize transport over the network.

  • /api/query/stockholm_pfam/paz
